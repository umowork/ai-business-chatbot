# AI Business Chatbot + Битрикс24/AmoCRM

> 💬 AI-чат-бот для бизнеса с **готовой интеграцией Битрикс24 / AmoCRM**. Не «AI-FAQ», а полноценный лидогенератор.

## Для бизнеса

**Что это:** Telegram-бот (или веб-чат), который ведёт диалог с клиентом, отвечает по бизнес-данным, квалифицирует лида и **автоматически создаёт сделку в CRM** с правильным менеджером.

**Категория спроса:** на FL.ru заказы «Создать бота для автосервиса», «Нейроэкзаменатор сотрудников на Битрикс24 + GigaChat». По vc.ru: бот с CRM-интеграцией — **20 000–100 000 ₽**, голосовой — до 200 000 ₽.

**Кому пригодится:**
- 🦷 Стоматологиям, салонам, клиникам — приём и квалификация заявок 24/7
- 🚗 Автосервисам — запись на ТО, диагностика по описанию проблемы
- 🏠 Агентствам недвижимости — первичная консультация
- 💼 B2B-услугам — квалификация и передача горячих лидов
- 🏫 Образовательным центрам — приём заявок и подбор курса

**Что получают:**
- ✅ Бот понимает контекст бизнеса (промпт + база ответов)
- ✅ Квалификация лидов по критериям клиента
- ✅ Авто-создание сделки в Битрикс24 / AmoCRM
- ✅ Передача горячих лидов менеджеру в Telegram
- ✅ Запись на услугу через интеграцию с календарём
- ✅ Поддержка GigaChat / YandexGPT (без VPN)

**Цена внедрения:** 20 000 — 100 000 ₽

## Для разработчика

**Стек:**
- Python 3.12 + **aiogram 3.13** (Telegram) + **FastAPI** (веб-чат, webhooks)
- PostgreSQL для диалогов и состояний
- Redis для FSM
- **OpenAI GPT-4o-mini** + **GigaChat** + **YandexGPT** через единый gateway
- **Битрикс24 REST API**
- **AmoCRM API**
- **Google Calendar API** (для записи на услугу)
- `instructor` для extraction полей из диалога

**Архитектура:**
```
[TG / Web Widget]
        ↓
[aiogram / FastAPI handler]
        ↓
[Dialog Manager]
   ├─ Intent detection
   ├─ Context (промпт + база ответов + история)
   └─ Lead Qualifier (Pydantic schema)
        ↓
[LLM Gateway: GigaChat / GPT / Yandex]
        ↓
   ┌────┴────────────┬──────────────┐
   ↓                 ↓              ↓
[Response]    [Lead в CRM]    [Push менеджеру]
              (Bitrix24 /    (TG-уведомление
              AmoCRM REST)    с резюме лида)
```

**Целевые метрики (замерим при реализации):**
- Доля успешно квалифицированных лидов
- Доля сделок созданных в CRM
- Latency ответа
- Cost per диалог
- Конверсия в передачу менеджеру

**Фичи (реализовано):**
- Конфиг бизнеса через `.env` + Pydantic Settings (имя, тон, FAQ-RAG)
- Lead qualification через LLM + Pydantic-схему (`{name, phone, service, urgency, budget}`)
- Auto-создание сделки/контакта в Битрикс24 / AmoCRM (`crm/` пакет с base / bitrix24 / amocrm / mock-адаптерами)
- Запись на услугу через Google Calendar API
- Intent classifier + RAG по бизнес-данным
- Передача горячих лидов в Telegram-чат менеджеров

**В Roadmap (см. [ROADMAP.md](./ROADMAP.md)):**
- YAML-конфиг бизнеса (сейчас через `.env`)
- Multi-tenant деплой (один сервис → несколько клиентов)
- Эскалация на человека по ключевым словам

**Что демонстрирует:**
- Интеграция Битрикс24 / AmoCRM в production
- Lead qualification через LLM + Pydantic
- Multi-channel (TG + Web widget)
- Multi-LLM gateway с РФ-провайдерами first-class
- Бизнес-логика (не только chat)
